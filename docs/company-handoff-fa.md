# تحویل کار برای ادامه روی سیستم شرکت

این سند برای ادامه کار روی کامپیوتر شرکت و راه‌اندازی فاز اول AGTPS Portal نوشته شده است.

## وضعیت فعلی پروژه

پرتال تا اینجا شامل این بخش‌هاست:

- صفحه اصلی پورتال RTL فارسی با ویجت‌های قابل تنظیم.
- پنل ادمین.
- مدیریت سامانه‌ها، دسته‌بندی‌ها و سایت‌ها.
- نقشه ایران فعلی به عنوان نسخه موقت.
- اخبار و اطلاعیه‌ها.
- تقویم جلسات، تسک‌ها، یادآوری‌ها و یادداشت‌ها.
- آموزش/LMS در فاز قابل قبول فعلی.
- فایل‌شیر سازمانی.
- وضعیت زنده سامانه‌ها با health check.
- نظرسنجی و رای‌گیری.
- مرکز اعلان‌ها و SMTP.
- بکاپ، زمان‌بندی بکاپ و Restore.
- Audit Log.
- ارتباط مستقیم کاربران با مدیران.

## تصمیم‌های مهم

- GitHub را Codex مستقیم push نمی‌کند. بعد از تایید کاربر، commit محلی زده می‌شود و دستور push به کاربر داده می‌شود.
- هر ماژول باید قابل فعال/غیرفعال شدن باشد.
- وقتی ماژول به حد قابل قبول رسید، توسعه همان ماژول متوقف و ایده‌ها به فاز بعدی منتقل می‌شود.
- هر بخش تکمیل‌شده باید هم در راهنمای فارسی و هم در مستند فنی ثبت شود.
- صفحه اصلی پورتال نباید پیام‌های ارتباط مستقیم را نشان دهد؛ فقط نوتیفیکیشن نمایش داده می‌شود.
- متن پیام‌های ارتباط مستقیم در دیتابیس با رمزنگاری سمت سرور ذخیره می‌شود.
- رمزنگاری فعلی End-to-End نیست و اگر سطح امنیتی بالاتر لازم شد باید فاز جداگانه طراحی شود.

## کارهای مهم بعد از clone در شرکت

1. ریپو را clone کنید.
2. فایل `docker/.env` را از روی `docker/.env.example` بسازید.
3. مقدارهای امن برای `JWT_SECRET` و `DIRECT_COMMUNICATION_ENCRYPTION_KEY` بگذارید.
4. سرویس‌ها را با Docker بالا بیاورید.
5. migration و seed را اجرا کنید.
6. وارد ادمین شوید و تنظیمات اولیه، SMTP، بکاپ، ماژول‌ها و کاربران را بررسی کنید.
7. Active Directory را در شرکت تست کنید.
8. فایل‌شیرهای واقعی را mount و در ادمین تعریف کنید.
9. سامانه‌های واقعی را برای Health Check ثبت کنید.

## مسیرهای مهم

- پورتال: `/`
- ورود ادمین: `/admin/login`
- داشبورد: `/admin/dashboard`
- تنظیمات: `/admin/settings`
- ماژول‌ها: `/admin/modules`
- مرکز اعلان‌ها: `/admin/notifications`
- مرکز بکاپ: `/admin/backups`
- Audit Log: `/admin/audit-logs`
- فایل‌شیر: `/admin/file-shares`
- پیام‌های مستقیم کاربر/مدیر: `/admin/direct-messages`
- تنظیمات ارتباط مدیران: `/admin/direct-communication`

## نکات محیط شرکت

- برای Active Directory، تست دقیق فقط داخل شبکه شرکت انجام شود.
- برای SMB، مسیرها باید روی Linux host mount شوند و داخل container از `/mnt/file-shares/...` تعریف شوند.
- برای Health Check سامانه‌ها، نوع check بر اساس نوع سامانه انتخاب شود: HTTP، TCP، Ping، SMB یا دستی.
- قبل از هر بروزرسانی production، از دیتابیس و فایل‌ها بکاپ بگیرید.
